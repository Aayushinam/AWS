AWS: AWS is a cloud provider that provides infrastructure as a service. AWS is the world's most comprehensive and broadly adopted cloud platform, offering over 200 featured services from data centers globally.


PUBLIC CLOUD:The services which are accessed by everyone like AWS.


PRIVATE CLOUD:The services which are accessed by the organization like an oracle.


HYBRID CLOUD:The combination of public and private cloud.


VERSIONING:Versioning is like a backup tool, versioning is not enabled on the bucket.


VIRTUALIZATION:Virtualization means creating a virtual version of a device or resources.


VM:The virtual machine is the virtualization/ emulation of a computer system.


EC2:Elastic compute cloud is a virtual server in amazon. Ec2 is regional.


REGION:It is a geographical area. A region has multiple data centers or availability zones.


AVAILABILITY ZONE:AZ is simply a data center. AZ can communicate with each other.


KEYPAIR:Keypair is used to retrieve the password of the ec2 instance


SECURITY GROUP: SG acts as a firewall to secure ec2 instances.


TYPES OF IPs:

PUBLIC IPs: which are accessed from the internet.

PRIVATE IPs: which can’t be accessed from the internet

ELASTIC IPs: An Elastic IP address is a static public IPv4 address associated with your AWS account in a specific Region.

LOAD BALANCER: We use the load balancer to distribute the traffic to the server.


ELASTIC BEANSTALK: EBS is used for easy and quick deployment of applications in AWS. You have full control of the ec2 instance which are launched by the beanstalk.


LAMBDA: AWS Lambda is a compute service that let you run the code without provisioning or managing servers


VIRTUAL PRIVATE CLOUD: VPC is a virtual network of data centers inside AWS for one client.


PUBLIC SUBNET:If subnet traffic is routed to an internet gateway is known as a public subnet.


PRIVATE SUBNET:If subnet traffic doesn’t route to an internet gateway is known as a private subnet.


INTERNET GATEWAY:An internet gateway is a horizontally scaled, redundant, and highly available VPC component that allows communication between your VPC and the internet


NAT GATEWAY (Network Address Translation service ):You can use a NAT gateway so that instances in a private subnet can connect to services outside your VPC but external services cannot initiate a connection with those instances.


AUTOSCALING: Create a group of ec2 instances that scale up or down depending on the condition you set.


AWS STORAGE:

  1. Simple storage service (s3)
  2. Elastic file system (EFS)
  3. Elastic block storage (EBS)
  4. Glacier
  5. Snowball


SIMPLE STORAGE SERVICE: S3 is storage for the internet, It has a simple web service interface for simple storing & retrieving of any amount of data, anytime from anywhere on the internet.


ELASTIC FILE SYSTEM: EFS can be mounted multiple ec2 instances at the same time across multiple AZs.


ELASTIC BLOCK STORAGE: You can attach multiple volumes to the ec2 instance. Volume can be attached & detached.


SNAPSHOT: The backup of the volume is called a snapshot.


IDENTITY AND ACCESS MANAGEMENT (IAM): You can control the entity AWS account using IAM by giving proper permissions to the IAM user.


IAM GROUP: Collection of IAM users.


IAM ROLES: Temporary access without credentials.


CLOUD FONT: CloudFront is a web service that speeds up the distribution of your static and dynamic web content, such as .html, .css, .js, and image files, to your users.


CLOUDWATCH: CloudWatch is a monitoring and observability service built for DevOps engineers, developers, site reliability engineers (SREs), IT managers, and product owners.


RELATIONAL DATABASE: RDS is not a database. It is a database service. RDS is a data structure that allows you to link information from the different tables of different types of data buckets


SIMPLE QUEUE SERVICE: SQS is a fully managed message queuing service that enables you to decouple and scale microservices, distributed systems, and serverless applications.


SIMPLE NOTIFICATION SERVICE: SNS is a fully managed messaging service for both application-to-application (A2A) and application-to-person (A2P) communication.


ROUTE 53: Amazon Route 53 is a highly available and scalable cloud Domain Name System (DNS) web service.
